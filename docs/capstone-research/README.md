# Capstone Research Documentation (EvaTrack)

Draft materials for the **Researcher** role. Copy into your `CAPSTONE.docx` and apply your school’s formatting (font, margins, page numbers).

## Files

| File | Content |
|------|---------|
| `00-RESEARCHER-ROLE-AND-CONTRIBUTIONS.md` | What a researcher does in a 4-person team |
| `CHAPTER-1-INTRODUCTION.md` | Background, problem, objectives, scope, significance |
| `CHAPTER-2-RELATED-LITERATURE.md` | Literature review + synthesis + sample references |
| `CHAPTER-3-METHODOLOGY.md` | SDLC, methods, tools, evaluation |
| `REQUIREMENTS-GATHERING.md` | Interview guide, survey, requirements list |
| `SYSTEM-ANALYSIS.md` | AS-IS / TO-BE, flows, stakeholder analysis |

## Quick steps

1. Open each `.md` file in VS Code or paste into Word.
2. Replace placeholders: barangay name, university, dates, respondent data.
3. Complete **real interviews** and update `REQUIREMENTS-GATHERING.md` Section 3.
4. Replace literature **placeholders** with PDFs from your library.
5. Share requirements + flows with backend/frontend teammates.
